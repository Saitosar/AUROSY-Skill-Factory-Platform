# AUROSY Skill Factory — web

Монорепозиторий **веб-клиента** платформы **AUROSY Skill Factory**. Код SPA: [`web/frontend/`](web/frontend/). Документация UI: [`docs/g1-control-ui/`](docs/g1-control-ui/). В приложении: главная как dashboard (`/`), настройки клиента и `VITE_API_BASE` — маршрут `/settings`; UI выровнен по дизайн-системе (полупрозрачные панели, общие компоненты заголовков и баннеров, тосты для сетевых ошибок — см. журнал F9 в `docs/g1-control-ui/03_implementation_roadmap_frontend.md`). Улучшения ключевых экранов и потоков (Авторинг, Pose Studio, сценарии, конвейер) — журнал **F10** в том же файле. **F11** — доступность: единое `:focus-visible`, бейджи статусов конвейера (иконка + текст), `prefers-reduced-motion` для тостов и индикаторов, улучшенные aria у слайдеров телеметрии — журнал в `docs/g1-control-ui/03_implementation_roadmap_frontend.md`. **F12** — пользовательский FAQ: [`docs/g1-control-ui/FAQ.md`](docs/g1-control-ui/FAQ.md) и встроенная справка в UI по маршруту **`/help`** (журнал F12 в том же roadmap). Интеграция **Phase 5** платформы (артефакты, очередь jobs, пакеты): клиент подставляет заголовок **`X-User-Id`** на соответствующие маршруты через **`apiFetch`** в [`web/frontend/src/api/client.ts`](web/frontend/src/api/client.ts); идентификатор задаётся в **Настройках**, переменной **`VITE_PLATFORM_USER_ID`** или fallback — см. [`web/frontend/README.md`](web/frontend/README.md). Экран **Задачи** (`/jobs`): сохранение JSON-артефактов, постановка асинхронного train и мониторинг статуса — журнал **F14** в [`docs/g1-control-ui/03_implementation_roadmap_frontend.md`](docs/g1-control-ui/03_implementation_roadmap_frontend.md). Экран **Пакеты** (`/packages`): Skill Bundle — список, скачивание, загрузка `.tar.gz`, публикация; упаковка из успешного job с деталей задачи — журнал **F15** в том же roadmap. **F16** — IA «Конвейер ↔ Платформа», перенос reference из preprocess на `/jobs` без автозапуска очереди — журнал **F16** в [`docs/g1-control-ui/03_implementation_roadmap_frontend.md`](docs/g1-control-ui/03_implementation_roadmap_frontend.md). **F17** — надёжность и продакшен: баннер при `telemetry_mode: dds` на экранах Телеметрии и Pose Studio (рекомендация mock до готовности DDS-моста), документация worker/`job_timeout_sec`/CORS, чеклист [`docs/g1-control-ui/DEPLOYMENT.md`](docs/g1-control-ui/DEPLOYMENT.md) и разделы FAQ/справки **`/help`** — журнал **F17** в том же roadmap. Бэкенд — отдельный репозиторий; связка при разработке описана в [`web/README.md`](web/README.md).
