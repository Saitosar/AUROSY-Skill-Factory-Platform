# Чеклист развёртывания (веб-UI + бэкенд)

Краткий список для согласованного опыта пользователей (фаза **F17** дорожной карты). Детали HTTP/WebSocket и переменных окружения — в README и OpenAPI **репозитория бэкенда**; здесь — точки сопряжения с этим монорепозиторием (`web/frontend/`).

## Перед выдачей в продакшен

- [ ] **Один origin для UI и API** (рекомендуется): reverse-proxy отдаёт статику SPA и проксирует `/api` и `/ws` на процесс FastAPI — меньше проблем с CORS и WebSocket.
- [ ] **CORS**: в dev часто `allow_origins=["*"]`; в продакшене сузить до доверенных хостов UI. См. настройки бэкенда.
- [ ] **`VITE_API_BASE`**: при сборке фронта задать URL API без завершающего `/` или оставить пустым при same-origin за прокси. WebSocket строится от того же хоста/схемы (`telemetryWebSocketUrl` в `src/api/client.ts`).
- [ ] **Прокси WebSocket**: убедиться, что путь `/ws/telemetry` (или полный префикс `/ws`) пробрасывается с теми же правилами, что и `/api`.
- [ ] **Phase 5**: заголовок `X-User-Id` согласован между UI (**Настройки**, `VITE_PLATFORM_USER_ID`, fallback) и бэкендом (`G1_DEV_USER_ID` в dev и т.д.). Вызовы идут через `apiFetch`.
- [ ] **Очередь jobs**: включён фоновый worker (`G1_PLATFORM_WORKER_ENABLED` или эквивалент в вашей сборке бэкенда), иначе задачи не выполняются.
- [ ] **Таймауты job**: на бэкенде может быть лимит длительности (часто `job_timeout_sec` или имя из конфига) — долгие train могут обрываться по таймауту; уточнить в документации сервера. Если `GET /api/meta` отдаёт `job_timeout_sec` / `platform_worker_enabled`, они могут отображаться в **Настройках** UI.
- [ ] **Телеметрия**: при `telemetry_mode: dds` (часто `G1_USE_DDS_TELEMETRY`) без развёрнутого DDS-моста WebSocket может нестабильно работать — для демо и разработки UI используйте **mock-телеметрию** на бэкенде до готовности моста (см. [FAQ.md §5](FAQ.md)).
- [ ] **Credentials**: текущий фронтенд не использует `fetch(..., { credentials: 'include' })`. При введении cookie-сессий на бэкенде потребуется согласовать CORS и клиент (`src/api/client.ts`).
- [ ] **MuJoCo WASM (Pose Studio):** статика `public/mujoco/g1/` (XML + STL) и собранные `*.wasm` из `@mujoco/mujoco` отдаются как обычные статические файлы. Для продакшена задайте **долгий `Cache-Control`** для `*.wasm` и тяжёлых мешей (например `immutable`, `max-age` от недели и выше), чтобы повторные визиты не перекачивали мегабайты.
- [ ] **Многопоточная сборка MuJoCo (`@mujoco/mujoco/mt`):** если когда-либо переключитесь на MT-сборку, для `SharedArrayBuffer` обычно нужны заголовки **`Cross-Origin-Opener-Policy: same-origin`** и **`Cross-Origin-Embedder-Policy: require-corp`** на том же origin, что отдаёт SPA (и согласованная политика для всех встраиваемых ресурсов). Текущая интеграция в репозитории использует **однопоточный** входной путь — эти заголовки **не обязательны**, пока не включён MT.
- [ ] **Черновики поз (опционально):** UI может вызывать `POST /api/platform/pose-drafts` (Phase 5, заголовок `X-User-Id` через `apiFetch`). Убедитесь, что маршрут включён на бэкенде и CORS разрешает origin UI, если API на отдельном хосте.
