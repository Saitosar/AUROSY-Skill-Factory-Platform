## 1. Доступность API и бэкенда

**Как понять, что бэкенд доступен?**  
На **Главной** отображается статус «API доступен». При недоступности показывается баннер вверху страницы и кнопка «Повторить» для повторной проверки `GET /api/health`.

**Что такое `VITE_API_BASE`?**  
Переменная окружения **сборки** фронтенда: базовый URL, на который уходят все запросы к REST и от которого строится WebSocket. Если не задана, браузер использует тот же origin, что и страница (в разработке Vite **проксирует** `/api` и `/ws` на машину с FastAPI). Эффективное значение видно в **Настройках** (`/settings`).

**Зачем `X-User-Id` и где его настроить?**  
Для маршрутов **Phase 5** (`/api/platform/*`, `/api/jobs*`, `/api/packages*`) бэкенд определяет владельца ресурсов по заголовку `X-User-Id`. Эффективное значение показано в **Настройках**: можно задать переопределение в браузере (localStorage), либо использовать **`VITE_PLATFORM_USER_ID`** при сборке, либо встроенный fallback (`local-dev`). HTTP-вызовы Phase 5 в коде должны идти через **`apiFetch`** в `src/api/client.ts`.

## 2. Авторинг: keyframes, motion, scenario

**Что редактируется на экране «Авторинг»?**  
Три типа артефактов Phase 0: **keyframes**, **motion**, **scenario** — как JSON. Есть загрузка golden-фикстур из `public/fixtures/golden/v1/` и кнопки форматирования/копирования.

**Где описаны форматы?**  
В репозитории бэкенда — каталог контрактов (см. [backend_references.md](backend_references.md)). Копии JSON Schema для офлайн-проверки лежат в `web/frontend/public/contracts/authoring/`.

## 3. Клиентская и серверная валидация

**Две кнопки проверки — в чём разница?**  
- **JSON Schema (клиент)** — проверка в браузере по локальной копии схемы (AJV). Быстро, работает без сервера.  
- **Сервер Phase 0** — `POST /api/validate`: источник правды для продакшн-логики на бэкенде.

**Почему результаты могут различаться?**  
Схема на клиенте могла устареть относительно сервера. После обновления бэкенда скопируйте актуальные `.schema.json` в `public/contracts/` и пересоберите фронт.

## 4. Pose Studio и ассеты

**Что за вкладка «WASM» / 3D в Pose Studio?**  
Клиентская симуляция через официальный пакет [`@mujoco/mujoco`](https://www.npmjs.com/package/@mujoco/mujoco) и MJCF робота **Unitree G1** из репозитория [mujoco_menagerie](https://github.com/google-deepmind/mujoco_menagerie/tree/main/unitree_g1). Модель и меши лежат в `public/mujoco/g1/`; для первичной загрузки ассетов разработчик запускает `npm run fetch:menagerie-g1`.

**Чем углы из WASM отличаются от WebSocket-телеметрии?**  
**Телеметрия** берёт углы из потока `/ws/telemetry` (живое состояние с бэкенда/робота или mock). **WASM** крутит модель **только в браузере**: слайдеры пишут в `qpos` MuJoCo, 3D обновляется локально, **без** WebSocket. Экспорт keyframes в Авторинг в обоих режимах использует **одни и те же** строковые имена суставов Phase 0.

**Почему первый заход в WASM «тяжёлый»?**  
Браузер один раз загружает WASM-рантайм MuJoCo (порядка мегабайт) и файлы сцены/STL. Их имеет смысл кэшировать на CDN/reverse-proxy.

Подробнее: [../mujoco-wasm-browser.md](../mujoco-wasm-browser.md).

## 5. Телеметрия и WebSocket

**Чем «Телеметрия» отличается от Pose Studio?**  
**Телеметрия** — мониторинг потока углов (таблица/слайдеры), без центральной схемы робота. **Pose Studio** — подбор позы: схема по группам суставов; углы можно брать из WebSocket или из локального MuJoCo WASM.

**Соединение рвётся или не устанавливается**  
Проверьте доступность бэкенда и что URL WebSocket согласован с `VITE_API_BASE` (тот же хост/схема). В UI есть индикация состояния и повторное подключение.

**Что значит режим DDS (`telemetry_mode: dds`)?**  
Бэкенд нацелен на поток телеметрии через DDS. Если DDS-мост не реализован или не запущен, `/ws/telemetry` может сразу закрываться. Для типичной разработки и демо UI переключите сервер на **mock-телеметрию**. На экранах **Телеметрия** и **Pose Studio** при режиме `dds` показывается предупреждающий баннер.

## 6. Сценарии и длительность

**Что значит предупреждение про 25–35 секунд?**  
Оценка суммарной длительности сценария (`POST /api/scenario/estimate`) вышла за ориентировочное «окно» для ролика. Это не жёсткий запрет — подсказка скорректировать цепочку нод или параметры.

## 7. Конвейер и CLI

**Зачем экран показывает результат `GET /api/pipeline/detect-cli`?**  
Бэкенд проверяет, видны ли в `PATH` команды `preprocess` / `playback` / `train`. Если нет — возможен fallback через `python -m ...`.

**Где смотреть ошибки preprocess / playback / train?**  
В блоках вывода на экране «Конвейер»: `exit_code`, stdout, stderr; длинные JSON могут быть усечены с кнопкой «Показать полностью».

## 8. Платформа: артефакты и очередь train

**Зачем сохранять артефакты через Phase 5?**  
Крупный JSON (например `reference_trajectory` после preprocess) удобнее один раз сохранить на платформе под именем (`POST /api/platform/artifacts/{name}`) и ссылаться на него из `POST /api/jobs/train`.

**Чем конвейерный train отличается от задачи в очереди?**  
**Конвейер** запускает train как синхронный subprocess (`POST /api/pipeline/train`) — результат сразу в ответе. **Задачи** — асинхронная очередь (`POST /api/jobs/train`): изоляция workspace и лимиты параллелизма задаются на бэкенде.

**Как перенести reference из preprocess в очередь train?**  
После успешного preprocess на экране **Конвейер** нажмите «Перейти к очереди обучения» — откроется `/jobs` с предзаполненным `reference_trajectory`. Постановка в очередь **не** выполняется автоматически.

**Задача не выходит из очереди — что проверить?**  
На бэкенде должен работать обработчик очереди (часто переменная `G1_PLATFORM_WORKER_ENABLED`). Без worker задачи не выполняются.

**Как выпустить Skill Bundle после успешного train?**  
На экране деталей задачи (`/jobs/:jobId`), когда статус успешный, доступна кнопка упаковки — запрос `POST /api/packages/from-job/{job_id}`.

**Где список пакетов, скачивание и загрузка?**  
Экран **Пакеты** (`/packages`): `GET /api/packages` (таблица), `GET /api/packages/{package_id}/download` (архив), `POST /api/packages/upload` (multipart `.tar.gz`).

## 9. Продакшен и развёртывание

**Чем dev отличается от продакшена по CORS?**  
В разработке бэкенд часто разрешает все origins (`allow_origins=["*"]`). В продакшене список нужно сузить до доверенных хостов UI. Надёжная схема — отдавать SPA и проксировать `/api` и `/ws` с одного origin (reverse-proxy).

**Нужны ли `credentials` во фронтенде?**  
Текущий клиент вызывает `fetch` без `credentials: 'include'`. Phase 5 опирается на заголовок `X-User-Id`, а не на cookie-сессию.

**Где полный чеклист?**  
Файл [../deployment/README.md](../deployment/README.md) — чеклист и инструкция по Vercel.

## 10. Язык интерфейса (i18n)

**Как переключить русский и английский?**  
Переключатель **RU / EN** в сайдбаре и на странице **Настройки**. Выбор сохраняется (в т.ч. `localStorage`), язык документа (`<html lang>`) обновляется.
